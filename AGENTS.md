# H Memo 저장소 규칙

## 작업 방식

- 사용자에게는 항상 존댓말을 사용하고, 대화와 문서는 한국어를 중심으로 작성해 주세요.
- `gpt-5.6-sol` 모델은 기획과 오케스트레이션 등 핵심 판단에만 사용해 주세요. 구현과 리뷰는 `gpt-5.6-terra` 모델로 진행해 주세요.
- `gpt-5.6-sol` 또는 `gpt-5.6-terra` 모델이 subagent를 구성할 때는 subagent에 `gpt-5.6-luna` 모델만 사용해 주세요.
- `gpt-5.6-sol` 모델을 사용하는 작업은 구현을 시작하기 전에 기획 내용을 Markdown 문서로 남겨 주세요.

## 코드 구성

- 한 파일의 코드가 500줄 이상이 되면 기능별로 나누어 새 파일로 분리해 주세요. 파일 하나에 지나치게 많은 기능이나 코드가 쌓이지 않도록 관리해 주세요.

## 버전과 릴리스

- 자동 릴리스 커밋이 아닌 실제 `main` 변경의 CI가 성공할 때마다 자동으로 patch 버전을 1 올립니다.
- 자동 릴리스 커밋은 다음 자동 증가의 대상이 아니며, 같은 원본 커밋에 대해 버전이 다시 증가하지 않도록 유지해 주세요.
- 개발자가 `package.json`, lockfile, Tauri, Cargo 파일의 버전을 직접 수동 편집하지 않습니다. 반드시 `npm run version:bump`을 사용합니다.
- 버전 변경 전후에는 `npm run check:versions`를 실행해 정합성을 확인합니다.

## 앱 변경 기록

- 웹앱에는 사용자가 확인할 수 있는 `업데이트 내역` 버튼을 적절한 위치에 유지해 주세요.
- 웹 또는 데스크톱 앱의 사용자 경험을 수정할 때마다 `업데이트 내역`에 개발일 또는 개선일과 간단한 변경 내용을 추가해 주세요.
- 웹 랜딩 페이지와 데스크톱 앱의 업데이트 내역은 같은 변경 사항을 사용자가 확인할 수 있도록 함께 갱신하고 테스트합니다.
