# Windows 실기기 스모크 테스트 체크리스트

이 문서는 Windows 설치 후 수동 검증에 필요한 항목을 정리합니다.
현재 작업 환경이 macOS인 경우, 아래 항목 중 **설치/트레이/부팅 동작**은 미검증으로 남겨두고 Windows 환경(실기기 또는 Windows VM)에서만 수행하십시오.

## 빠른 검증 절차

1. 릴리스 또는 CI에서 Windows 번들 아티팩트가 생성되었는지 확인합니다.
2. `npm run verify:windows-artifacts -- --bundle-dir <path>`로 출력 파일 존재를 먼저 점검합니다.
3. 설치 대상 Windows에서 체크리스트 항목을 실행해 증빙(스크린샷/로그/상태)을 수집합니다.

기본 번들 경로:

- `apps/desktop/src-tauri/target/release/bundle/msi/*.msi`
- `apps/desktop/src-tauri/target/release/bundle/nsis/*.exe`

권장 점검 경로(Windows CI/로컬 빌드 완료 후):

- `apps/desktop/src-tauri/target/release/bundle`

## 실행 체크리스트

### 사전 정보

- [ ] Windows 버전(예: Windows 11 23H2, build 번호)
- [ ] 하드웨어/VM 유형(실기기/VM/스냅샷 이름)
- [ ] 테스트 대상 설치 파일: `*.msi` 또는 `NSIS *.exe` 중 1개
- [ ] 앱 버전(설정/제목/파일명에서 확인)
- [ ] 설치 전 사용자 계정 상태(표준 사용자/Admin)
- [ ] 테스트 날짜 및 검사자

### 설치 및 실행

- [ ] 빌드 산출물 다운로드 (GitHub Release 또는 CI artifact)
- [ ] 설치 파일(다운로드 완료 시간 포함) 해시 또는 파일명 기록
- [ ] 설치 실행 후 설치 경로 및 완료 메시지 확인
- [ ] 앱 실행 확인(실행 후 바로 창이 열리지 않아도 트레이로 동작 가능한지 확인)
- [ ] 시작 직후 memo 창 또는 트레이 아이콘 동작 확인
- [ ] 시스템 트레이에 빈 사각형이 아니라 노란 메모 모양의 H Memo 아이콘이 노출되는지 확인

### 트레이 동작

- [ ] 트레이 아이콘 더블클릭(또는 지정된 클릭 동작)으로 memo 창이 열리거나 기존 창이 포커스되는지 확인
- [ ] 트레이 메뉴(있다면)에서 앱 열기/종료 동작이 정상인지 확인
- [ ] 트레이 아이콘이 앱 종료/종료 중복 실행 후에도 중복 생성되지 않는지 확인

### 메모 기능

- [ ] 새 메모 생성
- [ ] 새 메모를 만들면 기존 메모는 기존 창에 유지되고, 새 메모가 독립 창으로 열리는지 확인
- [ ] 여러 메모 창을 서로 다른 위치로 이동한 뒤 앱 재실행 후 위치/크기가 유지되는지 확인
- [ ] 앱 메뉴의 `메모 관리`에서 여러 메모 목록/열기/삭제 동작 확인
- [ ] 메모 내용 입력 및 수정
- [ ] 색상 변경
- [ ] 폰트 변경
- [ ] 폰트 크기 변경
- [ ] 글자색 변경
- [ ] 변경 사항이 저장되는지 확인(재실행 후 데이터 유지)

### 백업/내보내기

- [ ] TXT 로컬 백업/내보내기 동작
- [ ] 내보낸 파일 경로/이름 및 인코딩 확인
- [ ] JSON 로컬 백업 동작(여러 메모가 포함되는지 확인)
- [ ] JSON 로컬 복원 동작(복원 전 확인창과 여러 메모 재표시 확인)
- [ ] Firebase Google login/backup/restore
  - [ ] 설치한 빌드에 `VITE_GOOGLE_OAUTH_CLIENT_ID`가 포함되어 앱 메뉴의 계정 영역에서 `구글 로그인` 버튼이 별도 설정 입력 없이 활성화됨
  - [ ] 메모 상단바 종료 버튼 왼쪽의 Google 상태 아이콘이 로그인 전 회색/미로그인 상태로 표시됨
  - [ ] `구글 로그인` 클릭 시 시스템 기본 브라우저가 열리고 Google 계정 선택 후 앱으로 돌아오라는 완료 페이지가 표시됨
  - [ ] 앱 실행 후 `구글 로그인` 성공 표시가 보이고 `서버 백업` / `서버 복원` 버튼이 활성화됨
  - [ ] 로그인 후 상단바 Google 상태 아이콘이 로그인됨 상태로 바뀜
  - [ ] 여러 메모를 만든 뒤 `서버 백업` → `백업 완료:` 메시지 노출
  - [ ] 메모 내용 변경/삭제 후 `서버 복원` → 백업된 여러 메모가 다시 노출됨
  - [ ] 실패 시 에러 메시지와 로그 캡처

### 시작 등록(자동 실행)

- [ ] 시작 등록 토글 ON 동작 확인
- [ ] Windows 시작 시 앱 자동 실행 여부 확인
- [ ] 시작 등록 토글 OFF 동작 확인
- [ ] 사용자 로그아웃/리부팅 후 시작 등록 동작 확인(가능한 환경에서만)

### 종료/재설치

- [ ] 일반 종료 후 재실행 동작 확인
- [ ] 앱 제거(설치 프로그램 uninstall) 실행
- [ ] 제거 완료 후 잔존 프로세스/스타트업 등록/단축아이콘 잔여물 여부 확인
- [ ] 제거 후 재설치 가능 여부 확인

### 증빙 수집

- [ ] 트레이 아이콘/메모 창/백업 화면 스크린샷
- [ ] 설치 전·후 로그(가능 시 이벤트 로그, 앱 로그, 터미널 로그)
- [ ] 테스트 항목별 Pass/Fail 상태
- [ ] 실패 항목 시 재현 절차 및 재시도 결과
