# H Memo 홍보 페이지 최신 릴리스 표시 계획

## 목표

- 홍보 페이지의 대표 버전을 GitHub 최신 Release의 `tag_name`으로 표시합니다.
- 최신 Release 조회와 Windows 설치 파일 조회를 한 번의 API 요청으로 처리합니다.
- 최신 Release API가 실패하면 현재 웹 패키지 버전을 표시해 빈 상태를 방지합니다.
- 업데이트 기록에 v1.0.1의 주요 변경 사항을 추가합니다.

## 구현

1. `releaseDownload.ts`에 최신 버전과 MSI/EXE 상태를 함께 반환하는 함수를 추가합니다.
2. GitHub 응답의 `tag_name`을 검증하고, 잘못된 값은 표시하지 않습니다.
3. `LandingPage.tsx`가 통합 조회 결과를 사용하도록 변경합니다.
4. v1.0.1의 KST 일별 백업, KST 날짜 표시, 자동 버전·릴리스 내용을 업데이트 기록에 추가합니다.
5. API 성공, API 실패, 잘못된 버전 응답을 단위 테스트로 검증합니다.

## 검증

- 웹 단위 테스트
- TypeScript 검사와 프로덕션 빌드
- `/h-memo-releases/` 기준 빌드 결과에서 최신 Release API 및 대체 버전 확인
- 실제 홍보 페이지에서 표시 버전과 다운로드 링크 확인
