# H Memo 백업 기록 페이지네이션 계획

## 목표

- 서버 복원의 백업 기록 선택 목록을 페이지당 최신 10개로 제한합니다.
- 기록이 10개를 넘으면 이전·다음 버튼으로 나머지 기록을 확인합니다.
- 전체 기록 수, 현재 페이지, 전체 페이지 수를 함께 표시합니다.
- 데스크톱과 웹에서 동일한 동작과 스타일을 제공합니다.

## 구현

1. 공용 `BackupHistoryDialog`에 페이지 상태와 10개 단위 목록 분할을 추가합니다.
2. 대화상자를 새로 열거나 기록 목록이 바뀌면 유효한 첫 페이지를 보여줍니다.
3. 첫 페이지의 이전 버튼과 마지막 페이지의 다음 버튼을 비활성화합니다.
4. 데스크톱·웹 CSS에 페이지 이동 영역을 추가합니다.
5. 데스크톱 앱과 웹 홍보 페이지의 업데이트 내역에 개선 내용을 기록합니다.

## 검증

- 25개 기록에서 첫 페이지 10개, 두 번째 페이지 10개, 마지막 페이지 5개 확인
- 이전·다음 버튼 경계와 페이지 표시 확인
- 대화상자 재오픈 시 첫 페이지 복귀 확인
- 공용 UI 테스트, 데스크톱·웹 통합 테스트, 타입 검사와 프로덕션 빌드
